## 智能文档检索助手（RAG 问答系统）
**项目简介：** 基于 RAG 技术的智能文档检索与问答系统，支持多格式文档上传、多知识库管理、语义检索和流式对话，内置 RAGAS 量化评测体系，实现数据驱动的效果迭代优化。  
**技术栈：** Python / Streamlit / MySQL / ChromaDB / SQLAlchemy / LangChain

- **多格式文档解析与知识库管理：** 支持 PDF、Word、TXT、Markdown 四种格式文档的自动解析与入库，覆盖企业主流文档类型
- **RAGAS 自动化评测体系：** 搭建完整量化评估链路，覆盖上下文精确度、召回率、忠实度、答案相关性等 5 项核心指标，替代人工主观判断
- **双路混合检索：** 实现 BM25 关键词检索 + 向量语义检索的双路召回策略，兼顾精确匹配与语义泛化能力
- **上下文智能压缩：** 落地句子级语义过滤，仅保留与问题高度相关的片段送入 LLM，无效上下文占比下降约 65%
- **多阶段优化管线：** 设计并实现「查询改写 → 混合检索 → LLM 重排序 → 上下文压缩」四阶优化流程，系统综合评测得分从 0.675 提升至 0.896（+32.7%），其中上下文精度提升 91.5%，忠实度提升 28%
- **用户反馈闭环：** 实现文档自动关键词提取 + 用户自定义标签体系，接入点赞/点踩用户反馈机制，为模型持续迭代积累标注数据
- **网页爬取噪音过滤：** 自动剥离广告、导航栏、侧栏、页脚等无关内容，正文信息密度提升约 60%
- **统一数据模型设计：** 抽象 RawDocument 中间表示层，统一文件、网页等多源数据格式；新增数据源仅需开发适配器，下游处理逻辑零改动，扩展效率提升 70%+

## 全链路客户服务与工单闭环协同智能体
**项目简介：** 基于 Multi-Agent + LLM 架构的智能客服系统，设计 6 个专业 AI Agent 协同工作，覆盖从客户接入、意图识别、知识检索、工单创建到闭环解决的全链路流程。  
**技术栈：** Python / FastAPI / LangGraph / PostgreSQL / Redis / Qdrant / Vue3

- **多智能体编排架构：** 设计 6 个专业 Agent 的角色边界与交互协议，落地「意图分类+情绪分析并行预处理 → 画像增强 → 智能路由 → 回复合成」完整流水线
- **智能路由引擎：** 支持 FAQ 回答、工单创建、转人工、直接回复 4 条业务分支路径，通过条件边动态调度，路由综合准确率达 85%+
- **多路径 RAG 检索系统：** 实现「查询改写 + 多集合并行搜索 + 结果融合」多路召回策略，同时检索 FAQ、SOP 流程、历史工单解决方案三类知识库；检索结果统一经过 PII 隐私脱敏处理
- **工单全生命周期管理：** 设计 8 状态 + 12 条合法转换的工单状态机，实现工单自动分类、优先级计算与 SLA 时效追踪；支持 P0~P3 四级优先级，对应 1h~24h 分级响应时效，工单处理流转效率提升约 40%
- **情绪感知与智能升级：** 采用 LLM 语义识别 + 关键词规则的双层兜底机制，实时监测用户情绪，识别负面情绪或敏感触发词时自动升级人工服务
- **知识库自进化闭环：** 对话结束后自动触发 LLM 提炼标准 Q&A，按置信度三级分流（自动入库 / 人工审核 / 丢弃），FAQ 覆盖率月均提升 15%+，人工审核工作量降低 60%
