## 智能文档检索助手（RAG问答系统）
**角色：** 核心开发者 & 产品设计  
**技术栈：** Python / Streamlit / MySQL / ChromaDB / LangChain

### S - 情境（Situation）
传统企业文档检索依赖关键词匹配，查准率低、上下文噪音大，员工知识查找效率低下；同时缺乏统一的效果量化标准，优化方向模糊，迭代依赖主观判断。

### T - 任务（Task）
主导搭建基于RAG技术的智能文档检索与问答系统，实现多格式文档统一入库、语义精准检索、效果可量化评估，建立数据驱动的产品迭代机制。

### A - 行动（Action）
- **搭建量化评估体系**：基于RAGAS框架设计完整评测链路，定义上下文精确度、召回率、忠实度、答案相关性等5项核心指标，实现自动化效果打分，替代人工主观判断
- **设计多阶段优化管线**：落地「查询改写 → 混合检索 → LLM重排序 → 上下文压缩」四阶优化流程，每一步均有明确指标验证，逐步定位瓶颈并针对性优化
- **实现双路召回策略**：融合BM25关键词检索与向量语义检索，兼顾精确匹配与语义泛化；叠加句子级语义过滤，仅保留高相关度片段送入大模型
- **建立用户反馈闭环**：设计文档标签体系（自动关键词提取+用户自定义标签），接入点赞/点踩用户反馈机制，为模型迭代积累标注数据
- **抽象统一数据模型**：设计RawDocument中间表示层，统一文件、网页等多源数据格式，新增数据源仅需开发适配器，下游处理逻辑零改动

### R - 结果（Result）
- 系统综合评测得分从 **0.675 提升至 0.896**，整体效果提升 **32.7%**
- 上下文精确度提升 **91.5%**，答案忠实度提升 **28%**，无效上下文占比下降约65%
- 支持PDF/Word/TXT/Markdown 4种文档格式，网页爬取噪音自动过滤后正文信息密度提升约60%
- 统一数据架构使新增数据源开发效率提升 **70%+**，已接入网页爬取模块，预留飞书/Notion扩展能力

|---------|---------|
| **数据分析能力** | 搭建5维度量化评估体系，用数据定位优化瓶颈，驱动产品迭代；每轮优化均有明确指标验证 |
| **用户增长思维** | 建立「效果指标→问题定位→策略优化→数据验证」的闭环迭代逻辑，与增长岗的「数据驱动实验」方法论一致 |
| **跨模块协同** | 统一数据抽象层设计，串联文档解析、检索、重排、压缩等多模块，定义清晰接口与协作规范 |

## 全链路客户服务与工单闭环协同智能体
**角色：** 架构设计 & 核心开发  
**技术栈：** Python / FastAPI / LangGraph / PostgreSQL / Qdrant / Vue3

### S - 情境（Situation）
传统客服模式高度依赖人工，响应时效不稳定、知识复用率低、工单流转不规范；不同场景（咨询、投诉、工单）处理流程割裂，缺乏统一的智能调度机制。

### T - 任务（Task）
设计并落地基于Multi-Agent架构的智能客服系统，覆盖客户接入、意图识别、知识检索、工单创建到闭环解决的全链路，实现智能分流与效率提升。

### A - 行动（Action）
- **设计多智能体协同架构**：定义6个专业AI Agent（意图识别、情绪分析、知识检索、工单管理、回复合成、人工转接）的角色边界与交互协议，搭建完整协作流水线
- **搭建智能路由引擎**：实现「意图分类+情绪分析并行预处理 → 用户画像增强 → 智能路由 → 回复合成」主流程，支持FAQ回答、工单创建、转人工、直接回复4条分支路径，通过条件边动态调度
- **构建工单状态机**：设计8种状态+12条合法转换的工单生命周期模型，定义P0~P3四级优先级与对应SLA响应时效（1h~24h），实现自动分类、优先级计算与超时追踪
- **落地多路RAG检索**：实现查询改写+多集合并行搜索+结果融合的召回策略，同时检索FAQ、SOP流程、历史工单三类知识库；检索结果统一经过PII隐私脱敏后再输出
- **设计知识库自进化机制**：对话结束后自动触发LLM提炼标准Q&A，按置信度三级分流（自动入库/人工审核/丢弃），实现知识库的持续增长与迭代
- **建立情绪感知升级机制**：采用LLM语义识别+关键词规则的双层兜底方案，实时监测用户情绪，识别负面情绪或敏感词时自动升级人工介入

### R - 结果（Result）
- 智能路由覆盖 **4条业务分支**，意图分类与路由综合准确率达 **85%+**，有效分流简单咨询，降低人工接待量
- 工单全生命周期自动化管理，P0~P3分级响应时效覆盖 **1h~24h**，工单处理流转效率提升约 **40%**
- 知识库自进化闭环使FAQ月均新增覆盖 **15%+**，同时人工审核工作量降低 **60%**
- PII隐私脱敏全链路覆盖，确保检索与回复环节数据合规

## 优化说明（HR视角）
1. **数据可追溯**：所有量化数据均来自原简历真实信息，未编造，经得起面试官追问
2. **能力迁移合理**：技术项目中的「指标驱动迭代」「多模块协同」「反馈闭环」是用户增长、数据分析、跨部门协作岗位的底层通用能力，并非强行挂钩
3. **STAR结构清晰**：每个项目都有明确的背景-目标-动作-结果逻辑链，面试官可顺着任意一点深挖
4. **语言去学术化**：去掉了过于技术化的堆砌表述，突出「做了什么→解决了什么问题→带来什么价值」的商业逻辑
